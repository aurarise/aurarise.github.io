+++  

Perlin noise has **3** basic steps:  

**1**) Defining the grid  

**2**) Computing dot products  

**3**) Interpolation  
  
+++  
  
**Step 1: Defining the grid**

![Alt text](content/contentimgs/perlin5.png)

Each of the large grid tiles is a tile in our 2D array. I.e. It is one point of our noise. We will end up with hundreds of these in our final product. So, for each large tile, we need to select a point, like so:

![Alt text](content/contentimgs/perlin6.png)  

A point to note is that this grid size can be changed. We could have a grid where the total number of large tiles are 2x2, 4x4, 8x8, 16x16, etc. The total pixel size of the grid remains the same, therefore the more large tiles there are, the smaller they get. This increases how many samples are taken, and therefore the *scale* of the grid. More on this later.  

+++  

**Step 2: Computing the dot product**  

Anyway, we have defined our grid, next is computing dot products. It is highly recommended to research what a dot product is, assuming you don't know already. (https://betterexplained.com/articles/vector-calculus-understanding-the-dot-product/) but I will give a simple explanation here.  

The dot product is, in this case, geometric projection. It is a single value, that defines how aligned two vectors are with each other. E.g. take two input vectors, e.g. (1, 3) (2, 0) and calculate their dot product:  

a · b = (ax × bx) + (ay × by)

(There is another way, a · b = |a| × |b| × cos(θ))  

So to substitute:  

 a · b = (1 * 2) + (3 * 0)  

 = 3 + 0

 = 3  

A graphcal representation of our two vectors:  

![Alt text](content/contentimgs/perlin7.png)  


 **The larger the dot product, the more aligned the two vectors are, i.e. the angle between them is smaller, and vice versa.**  

 **A negative dot product means the vectors are pointing in opposite directions**

From the dot product of 3, we can calculate the angle between the vectors, as well as the direction (+ or -). In other words, this single value can represent the magnitude and direction of the combined vectors.  

So, applying this to our noise grid. As you can possibly guess, we need two vectors to calculate the dot products of each square tile in the grid. Furthermore, we need a dot product for **each corner** of the tile. Why this is needed will become apparent.  

How do we get two vectors for each corner? We need the **gradient vector** of the corner, and the **distance vector** of the corner to the target point.  

![Alt text](content/contentimgs/perlin9.png)  

This graphic displays all the vectors in our square tile. the gradient vectors are green, and the distance vectors are the orange. It is important to note that the gradient vectors can be in a completely random direction (although in improved Perlin noise, they are always pointing to cardinals and intercardinals as shown here.)    

+++  


Firstly to calculate the gradient vectors:
For each corner, they are simply the green arrows. They also have a nominal length of 1. For each corner, they are as follows:  

SW corner: (-1, 1)  
NW corner: (1, 0)  
NE corner: (0, 1)  
SE corner: (1, 1)  

Calculating the distance vectors are the orange arrows:

SW corner: (0.63 - 0, 0.39 - 0)  
 = (0.63, 0.39)  

NW corner: (0.63 - 0, 0.39 - 1)  
 = (0.63, -0.61)   

NE corner: (0.63 - 1, 0.39 - 1)  
 = (-0.37, -0.61)   

SE corner: (0.63 - 1, 0.39 - 0)  
 = (-0.37, 0.39)    
 
+++  

Using the dot product equation:    

        a · b = (ax × bx) + (ay × by)  

  therefore gives us 4 dot products, where a is the gradient vector, and b is the distance vector:  


SW corner:  (−1 * 0.63) + (1 * 0.39)  
            **= −0.24**     
NW corner:  (1 * 0.63 ) + (0 * −0.61)  
            **= 0.63**    

NE corner:  (0 * -0.37) + (1 * -0.61)  
            **=  −0.61**    

SE corner:  (1 * −0.37)+(1 * 0.39)  
            **= 0.02**  

These dot products represent the *strength* of each corner gradient in relation to point. **In other words, they tell much influence each corner has to the point - The closer the corner to the point, the more influence it has.** 

+++  

**Step 3: Interpolation**  

The final step. This means that we use linear interpolation on the dot products. The point of interpolation, is essentially a way to blend these influence values together:    

        Y = (a + x) * (b-a)  

a = dot product 1  
b = dot product 2  
x = special smoothing function of the original distance vector.  

 Since linear interpolation results in a straight line, it means that the final result would be sharp and not smooth. Therefore, we need a smoothing function to add a curve into the final line on the graph. There are many different options that are possible, but for Perlin noise, this one is commonly used: 

                x = 6t^5 - 15t^4 + 10t^3 

Where t is the original distance vector/coordinate for x and y  

**As an example, for the top axis: If the point is closer to the NW axis, then it would result in a more positive value. If it were closer to the NE axis, it would result in a more negative value.**   


We first interpolate the top X axis together, then the bottom X axis, and then both together for 3 total interpolations:


t = 0.63  
x = 6t5-15t4+10t3   
x = 0.7325  

Y = (0.63 + 0.7325) * (−0.61 - 0.63)  
        = −1.6895  

Next, we do the same for the bottom axis:  

Y = (−0.24 + 0.7325) * (0.02 - −0.24)      
        = 0.12805  

Lastly, our final interpolation. We simply interpolate these two values together (bilnearly interpolating them). Remember to use the y coordinate in the smoothing function this time:   

Y = (−1.6895 + 0.3005) * (0.12805 - −1.6895)  

        = −2.52457695  

+++  

And that is the final value, that can be normalised and then mapped to a colour. This is repeated for every square in the grid. It is important to note that the gradient vectors for each corner do not change. These are therefore the constants. Remember when I talked about scaling? We can adjust the small square size, but the gradient vector will still travel across these smaller squares from the larger corner:  

![Alt text](content/contentimgs/perlin10.png)    

As such, each adjacent square in the grid shares two gradient vectors (diagonal squares share one). This enables a smooth gradient between squares.
